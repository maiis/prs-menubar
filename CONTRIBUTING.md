# Contributing

Thanks for your interest in contributing!

## Workflow
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/short-description`
3. Make changes (keep them focused and minimal)
4. Run tests locally
5. Open a Pull Request describing the motivation and changes

## Commit Messages
- Use concise, present-tense messages (e.g., `fix: handle 401 auth error`)
- Focus on the *why* when possible

## Code Style
- Swift 6, strict concurrency
- Prefer clarity over cleverness
- Avoid adding dependencies unless essential

## Tests
- Add/adjust tests when modifying logic
- Keep tests fast and deterministic

## Security
- Never commit real tokens or secrets

## Questions
Open an issue for discussion before large refactors or new features.
